Documentation

Problem Statement

Procurement processes in many organizations still rely heavily on manual verification of Purchase Orders (PO), Goods Received Notes (GRN), and Invoices. This manual process is:

  • Time-consuming and labor-intensive
  • Prone to human error
  • Difficult to scale with increasing transaction volumes
  • Lacks real-time verification capability

Our solution automates this verification process using machine learning, reducing processing time from hours to seconds while improving accuracy.

Dataset Description

Due to the sensitive nature of procurement data, we created a synthetic dataset that mimics real-world procurement scenarios with the following characteristics:

Dataset Features

  • 50,000 synthetic transactions
  • PO, GRN, Invoice quantities
  • Vendor information
  • Item categories

Data Distribution

  • 70% match cases
  • 30% mismatch cases
  • Split into 80% training, 20% test
  • 5-fold cross validation

Model Description

We implemented a Random Forest classifier for this task due to its:

  • Ability to handle non-linear relationships
  • Robustness to outliers
  • Feature importance analysis capability
  • Good performance on small-to-medium datasets

The model achieves 92% accuracy on our test set with 89% precision in detecting mismatch cases.

Limitations & Future Work

Current Limitations

  • Uses only quantity data (no text analysis)
  • Trained on synthetic data
  • Doesn't integrate with ERP systems
  • Limited to three document types

Future Enhancements

  • Add NLP for document text analysis
  • Integrate with popular ERP systems
  • Expand to more document types
  • Implement anomaly detection
